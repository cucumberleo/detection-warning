# 气体监测 Web 平台重构评估

## 结论

原型已经证明了完整业务闭环，但原 `back/`、Flutter `frount/` 和单文件 Web demo 的职责重叠，接口命名也不一致。正式 Web 版本应先保证“监测可信、数据可追、预警可闭环”，再扩展推荐和 AI 助手。

本次重构采用 FastAPI + SQLite + Vue 3 + TypeScript：后端以平台模块统一持久化、角色可见范围和预警判定，前端按任务而不是技术模块组织信息架构。

## 现有功能盘点

| 来源 | 已实现能力 | 主要问题 |
|---|---|---|
| `back/` Flask | Excel 模拟、滤波/峰谷特征、浓度换算、SQLite 历史、历史摘要 | 业务、算法、数据库和 HTTP 全部集中在单文件；气体命名和阈值不统一；配置中存在明文密钥风险 |
| `frount/` Flutter | BLE 扫描连接、实时曲线、历史趋势、阈值预警、系统通知、CSV、中英双语 | `main.dart` 超过 2,000 行；固定后端 IP；移动端能力无法原样搬到通用浏览器 |
| 原 Web demo | 用户/管理员双入口、概览、模拟检测、预警、用户设备、推荐、AI 助手 | 数据只在内存；阈值按钮不可操作；无曲线和导出；单文件 JS 难维护；移动端隐藏主导航 |

## 功能优先级

### P0 - 必须形成闭环

- 设备选择、模拟/网关监测与实时响应曲线；
- 后端统一浓度与风险判定，结果持久化；
- 历史查询、趋势图、筛选与 CSV 导出；
- 预警阈值管理、预警生成和人工确认；
- 加载、失败、空状态以及输入校验；
- 角色可见范围，避免普通用户读取其他设备数据。

### P1 - 平台管理

- 设备在线状态、材料与位置；
- 用户和设备授权概览；
- 本地、可解释的历史摘要；
- 响应式布局和键盘可访问性。

### P2 - 有条件扩展

- 个性化推荐只有在真实行为数据和评价指标具备后才值得上线；
- LLM 助手必须通过权限受控工具读取数据，不能直接访问数据库；
- 浏览器直连 BLE 受安全上下文和浏览器兼容性限制，生产环境优先采用“设备/手机网关 -> 后端 -> WebSocket/SSE -> Web”链路；
- 系统通知和振动可在 PWA 权限模型明确后补充。

## 信息架构与布局评估

重构后的一级导航控制为六个以内：概览、实时监测、预警、历史分析、用户与设备（管理员）、AI 助手。这样对应用户任务，避免将“模型质量”“算法链路”等研发信息与业务入口并列。

- 桌面端使用固定侧栏和大面积工作区，表格与曲线拥有足够宽度；
- 移动端使用底部任务导航，不再简单隐藏导航；
- 预警使用琥珀/红色，正常状态使用绿色，颜色之外同时提供文字标签；
- 曲线、表格、表单均包含空状态、错误状态和清晰单位；
- 管理员阈值编辑与事件列表并置，便于理解配置对判定的影响。

## 模块与 seam

```text
Vue views -> typed HTTP client -> FastAPI routes
                                  |
                                  v
                      GasMonitorPlatform interface
                                  |
                                  v
                         PlatformStore adapter
                                  |
                                  v
                               SQLite
```

`GasMonitorPlatform` 是业务 seam：调用者只学习监测、概览、预警、导出和摘要接口，不接触 SQL。`PlatformStore` 是持久化 adapter，后续可替换为 PostgreSQL。真实算法只需替换检测生成实现并继续返回 `contracts/detection-result.v1.json`。

## 尚未声称完成的能力

- 当前曲线和浓度仍是可复现模拟，用于验证产品闭环，不应作为实验结果；
- 旧 Flask 的真实 Excel 信号处理尚未迁入新算法 adapter；
- Demo 登录不是生产认证，正式版需要 OIDC/JWT、服务端鉴权和审计日志；
- 旧 `back/config.json` 中的明文 API 凭证应立即作废并改为环境变量，不能提交到版本库。
